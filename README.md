# Focus

A focus timer for ADHD brains. Built around two coupled problems:

1. **Stuck** — can't start. The home offers an AI "Unstick me" path (you dump what's in your head, Claude returns one tiny first action) and three universally-useful physical actions (water · breaths · open the doc).
2. **Ready** — knows what to do, just wants the timer running. One-tap "Start now" → duration · tag · optional intent line → focus session with a draining-wedge timer.

Light/dark adapts to system setting. Single warm accent. Tick or lo-fi audio during focus, both generated by Web Audio API (no MP3 dependency). Local-storage only — no accounts, no sync, no analytics.

## Setup

```bash
cp .env.local.example .env.local
# edit .env.local and paste your Anthropic API key:
#   ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx
npm install
npm run dev
```

Then open http://localhost:3000.

> **Gotcha:** Next.js prefers shell env vars over `.env.local`. If you've already exported `ANTHROPIC_API_KEY` in your shell (Claude Code sets one as empty, for example), the file is overridden silently. Either `unset ANTHROPIC_API_KEY` first, or run `env -u ANTHROPIC_API_KEY npm run dev`.

## Architecture

| File | What |
|---|---|
| `app/page.tsx` | Single-page state machine — modes: `home`/`start`/`unstuck`/`primer`/`session`/`break`/`complete`/`stats`/`settings`. |
| `app/api/unstuck/route.ts` | Server-side Anthropic proxy. Reads `ANTHROPIC_API_KEY` from env. Never exposed to the client. |
| `lib/timer.ts` | Timer hook. Anchors against `Date.now() + durationMs` for accuracy across background-tab throttling, animates via `requestAnimationFrame`. |
| `lib/audio.ts` | Web Audio API — tick (sine burst once/second, scheduled ahead via `audioContext.currentTime`) and a soft warm pad for "lo-fi". |
| `lib/store.ts` | localStorage helpers + React hooks for tags/settings. |
| `lib/prompts.ts` | Fixed physical-action list and break-cue rotation pool. |
| `components/Wedge.tsx` | The draining circle hero element. CSS `conic-gradient` reactive to progress. |
| `components/Home.tsx` | Two-zone home (Stuck / Ready). |
| `components/StartSheet.tsx` | 4 duration presets + tag picker + optional intent. |
| `components/UnstuckSheet.tsx` | Textarea + AI suggestion display. |
| `components/Primer.tsx` | Big-emoji "Go do it" screen for the physical actions. |
| `components/SessionView.tsx` | Running session — wedge, time, tag, intent anchor, audio toggle. |
| `components/BreakView.tsx` | Recharge break with rotating cue. |
| `components/Complete.tsx` | Optional tag prompt + Done/Start another. |
| `components/Stats.tsx` | Totals view (this week / month / all time, by tag, no charts). |
| `components/SettingsView.tsx` | Audio mode, volume, default duration, tag CRUD (max 6). |

## Known limitations of web v1

- **Background tabs throttle.** A focus session backgrounded by the user works correctly on resume because the timer is anchored to absolute time, but `requestAnimationFrame` pauses (the wedge stops animating). The math is right; the visual just freezes until the tab is foregrounded.
- **Audio requires a user gesture to start.** Browsers won't auto-resume audio across tab backgrounding. The first tap on "Start" (or any button) is what unlocks the audio context.
- **iOS Safari is aggressive.** Long sessions may suspend if the user locks the phone or switches apps. Acceptable for a v1 web prototype; would need native iOS for reliability.

## What's intentionally NOT here (deferred)

- Accounts, login, cloud sync.
- Notifications.
- Streaks, daily goals, comparison stats.
- Charts in totals.
- Tag colors, projects, hierarchies.
- Apple Watch app.
- Hyperfocus protection (mandatory long break, escalating haptics).
- "Capture" parking-lot for mid-session intrusive thoughts.

These are tracked in `docs/superpowers/specs/2026-04-26-pomodoro-app-design.md` for the native iOS roadmap if that future returns.
